# 🤖 Telegram User Bot - Smart Auto Like

Умный Python бот, который автоматически ставит случайные реакции (👍 ❤️ 🐬) на сообщения с ключевыми словами в Telegram группах.

## ⚡ Возможности

- ✅ Мониторит групповой чат в реальном времени
- ✅ Ставит случайные реакции (👍 лайк, ❤️ сердце, 🐬 дельфин)
- ✅ Фильтрация по ключевым словам (лабораторная, работа, коллоквиум и т.д.)
- ✅ Фильтрация по ID пользователя (реагирует только на сообщения от конкретных людей)
- ✅ Легкий и быстрый код
- ✅ Работает через Docker

## 📋 Требования

- Docker и Docker Compose
- Аккаунт Telegram
- API ID и API HASH от Telegram

## 🚀 Быстрый старт

### 1️⃣ Получить API ключи Telegram

1. Перейти на https://my.telegram.org/auth/login
2. Войти со своего номера телефона
3. Перейти в "API development tools"
4. Скопировать `API ID` и `API HASH`

### 2️⃣ Получить ID группы

Отправьте сообщение боту @userinfobot в группе - он вышлет ID группы в формате `-1002584565460`

### 3️⃣ Получить ID пользователя

Отправьте сообщение боту @userinfobot и скопируйте **User ID**

### 4️⃣ Настроить проект

```bash
# Копируем .env.example в .env
cp .env.example .env

# Редактируем .env с вашими данными
nano .env
```

**Заполните .env файл:**
```env
TELEGRAM_API_ID=123456789
TELEGRAM_API_HASH=your_api_hash_here
TELEGRAM_CHAT_ID=-1002584565460
TELEGRAM_PHONE=+7XXXXXXXXXX
ALLOWED_USER_IDS=123456789,987654321
```

### 5️⃣ Первый запуск (авторизация вручную)

```bash
# Запустите в интерактивном режиме для авторизации
docker-compose run --rm telegram-bot
```

Введите код подтверждения из Telegram. Когда авторизация пройдёт успешно, бот выключится.

### 6️⃣ Запустить бота в фоне

После успешной авторизации запустите:

```bash
docker-compose up -d

# Проверить логи
docker-compose logs -f
```

## 📝 Переменные .env

| Переменная | Описание | Пример |
|-----------|---------|--------|
| `TELEGRAM_API_ID` | API ID от https://my.telegram.org | `123456789` |
| `TELEGRAM_API_HASH` | API Hash от https://my.telegram.org | `a1b2c3d4e5f6g7h8...` |
| `TELEGRAM_CHAT_ID` | ID группового чата для мониторинга | `-1002584565460` |
| `TELEGRAM_PHONE` | Номер телефона вашего аккаунта | `+79991234567` |
| `ALLOWED_USER_IDS` | ID пользователей для реагирования (через запятую) | `123456789,987654321` |

## 🎨 Как работает бот

1. **Мониторит группу** - следит за новыми сообщениями в указанной группе
2. **Проверяет отправителя** - реагирует только на сообщения от пользователей из `ALLOWED_USER_IDS`
3. **Проверяет ключевые слова** - ищет в сообщении:
   - лабораторная, лабораторка
   - сдача, работа
   - коллоквиум, типовик, типовая работа
   - экзамен, зачёт, проект
   - тест, отчёт, защита
   - исследовательская, практикум
4. **Ставит случайную реакцию** - выбирает одну из: 👍 ❤️ 🐬
5. **Логирует всё** - видите все события в консоли

## 🛠️ Полезные команды

```bash
# Запустить в интерактивном режиме (для авторизации)
docker-compose run --rm telegram-bot

# Запустить в фоне
docker-compose up -d

# Посмотреть логи в реальном времени
docker-compose logs -f

# Посмотреть последние 100 строк логов
docker-compose logs -f --tail=100

# Перезагрузить бота
docker-compose restart telegram-bot

# Остановить бота
docker-compose down

# Удалить контейнер и данные
docker-compose down -v

# Пересоздать контейнер
docker-compose up -d --force-recreate
```

## ⚙️ Как изменить ключевые слова

В `main.py` найдите список `KEYWORDS`:
```python
KEYWORDS = [
    'лабораторная',
    'лабораторка',
    'сдача',
    'работа',
    # ... добавьте свои
]
```

После изменения пересоберите:
```bash
docker-compose down
docker-compose build --no-cache
docker-compose up -d
```

## 🎲 Как изменить реакции

В `main.py` найдите список `REACTIONS`:
```python
REACTIONS = ['👍', '❤️', '🐬']
```

Замените на нужные эмодзи:
```python
REACTIONS = ['👍', '❤️', '🔥', '😂']
```

Пересоберите:
```bash
docker-compose down
docker-compose build --no-cache
docker-compose up -d
```

## 🚨 Решение проблем

### Бот не видит сообщения
- Проверьте ID чата: `TELEGRAM_CHAT_ID`
- Проверьте ID пользователя: `ALLOWED_USER_IDS`
- Посмотрите логи: `docker-compose logs -f`

### Нужно переавторизоваться
```bash
rm -rf sessions/
docker-compose run --rm telegram-bot
```

### Хочу обновить код на сервере
```bash
# Скачиваем последний код
git pull origin main

# Пересоздаём контейнер
docker-compose down
docker-compose build --no-cache
docker-compose up -d

# Проверяем логи
docker-compose logs -f
```
